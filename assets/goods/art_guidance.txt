===========================================
GOODS ASSET GENERATION - ART GUIDANCE
===========================================

STYLE: Close-up view of the good with textured background
SIZE: 1024x1024 pixels (standard)
BACKGROUND: Semi-transparent or textured (wood, cloth, stone)
AESTHETIC: Painterly, fantasy game art, DOTA-inspired
LIGHTING: Warm, inviting, showcases the good clearly

GENERAL NOTES:
- Each image should be a close-up that clearly shows the good
- Background should be textured (wood planks, cloth, stone table, etc.) but not distract from the item
- Use warm, painterly art style consistent with DOTA universe
- Items should look hand-crafted and fantasy-themed
- Consider faction aesthetics where relevant

===========================================
FILENAME CONVENTION
===========================================

All goods use snake_case matching the goodId in economy.js:
- grain.png
- corn.png
- fish.png
- iron_ore.png (not iron-ore.png or ironore.png)
- mana_crystals.png
- magic_items.png
- enchanted_armor.png
- fine_clothing.png

This ensures direct lookup: assets/goods/{goodId.toLowerCase().replace(' ', '_')}.png

===========================================
INTEGRATION NOTES
===========================================

Once generated, goods images can be used in:
1. Trading screen - show the good being bought/sold
2. Cargo manifest - visual inventory display
3. Port production view - show what's being made
4. Market price lists - icon next to good name
5. Tooltips - preview when hovering over good name

Loading in renderer.js:
```javascript
const goodsImages = {};
for (const goodId in GoodsData) {
    const filename = goodId.toLowerCase().replace(/\s+/g, '_');
    const img = new Image();
    img.src = `assets/goods/${filename}.png`;
    await img.decode();
    goodsImages[goodId] = img;
}
```
